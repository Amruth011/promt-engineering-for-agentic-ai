Prompting for Agentic AI

Why prompting an AI agent is completely different from prompting a chatbot — and how to get exactly what you imagine

Chatbot prompt

One question, one answer

You type. It replies. Done. The whole job is in one message. Like texting a friend.

Agent prompt

A job description

You define a goal. The agent figures out the steps, uses tools, checks results, and loops until it's done.

The big shift: A chatbot prompt answers "what should you say?" An agent prompt answers "what are you trying to achieve, what tools do you have, and what does done look like?"

Same request. Totally different prompts.

How an agent thinks through a task

An agent doesn't just reply — it loops. Every time it gets new information, it thinks again. Your prompt controls how it thinks at each step.

1

Understand the goal

Your system prompt tells the agent what it is and what it's trying to achieve. This is not "be an assistant" — it's "you are a booking agent whose only job is to find and confirm a restaurant table."

2

Plan the steps

The agent breaks the goal into sub-tasks. Good prompts tell it how to do this: "First check availability. Then confirm the date with the user. Then book."

3

Use a tool

It calls a tool — like searching availability or sending a message. Your prompt must describe what tools exist and when to use them. Vague tool descriptions = wrong tool calls.

4

Check the result

It reads what came back. Did it work? Is the info good enough? Your prompt controls what "good enough" means.

5

Loop or finish

If the result is good → done. If not → go back to step 2 and try again. Your prompt defines what "done" looks like — without this, agents loop forever or stop too early.

The prompt controls the loop. A chatbot prompt needs a tone and a task. An agent prompt needs: a goal, a stop condition, a list of tools, and rules for when things go wrong. Every missing piece = unpredictable behaviour.

The three things your prompt controls in an agent

1 — Tools

What the agent can do

Tools are like the agent's hands. Without a tool, it can only talk. With tools, it can search, book, email, read files, run code — anything.

Your prompt must say when to use each tool, not just that it exists. "Use search_web when you need facts from after 2023" beats "you have a search tool."

2 — Memory

What the agent remembers

Agents can remember things you told them earlier in the conversation — and with the right setup, across conversations too.

Your prompt can say: "Always check what the user has told you before asking again." This stops the agent from being annoying and repetitive.

3 — Multi-step decisions

How it handles complexity

Some tasks have multiple paths. Your prompt defines the decision rules — like an instruction manual for how to think.

Example: "If the first search returns nothing useful, try rephrasing the query before giving up. Never say 'I don't know' without trying at least twice."

A real agent prompt — annotated

# Who you are
You are a research assistant. Your job is to answer questions
using real sources — never from memory alone.

# Your tools
- search_web(query) → Use when you need current facts
- read_page(url) → Use to read a full article

# How to work
1. Search first. Read the top result.
2. If it doesn't answer the question, search again with
different words.
3. Try at least 2 searches before saying you can't find it.

# When you're done
Give a 3-sentence answer. Always say where you found it.
